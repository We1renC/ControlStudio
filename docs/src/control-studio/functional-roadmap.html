ControlStudio Functional Roadmap

後續開發（非 UI/UX）的核心方向、演算法規格、API 設計、驗收條件與依賴關係。 本文件為 canonical 工程規劃，任何接手 agent 須先閱讀本文。 目前 Tier A-J 已完成 deterministic baseline，full suite 基線為 111/111，fixture/API contract 為 7/7 cases。

Items: 57 Tiers: 10 (A–J) Estimated Total: ~270 工作天 MVP Path: 3 個月 Last Updated: 2026-06-18 Status: Implemented baseline Owner: Functional Planning

總覽 Overview

本路線圖將 ControlStudio 從「教學/驗證工具」進階為「設計-部署-驗證-硬體閉環」的完整工程平台。 現況基礎 (P9–P65) 已涵蓋線性控制、MPC、sysid、robust、模型降階等。 本文件規劃 57 項新功能/強化，分為 10 個 Tier，每項皆有獨立規格與驗收。

Tier A新增控制演算法 #

補齊現代控制設計矩陣中尚未實作的關鍵演算法族。

A1

ADRC — Active Disturbance Rejection Control

HIGH 3 天
目的

不需精確模型即可控制不確定/時變系統，以 ESO（擴展狀態觀測器）估測總擾動並前饋補償。產業熱門（台達/西門子產線）。

演算法核心
ESO（三階形式）：
  ẑ₁̇ = ẑ₂ − β₁(ẑ₁ − y)
  ẑ₂̇ = ẑ₃ − β₂·fal(ẑ₁−y, α₁, δ) + b₀·u
  ẑ₃̇ = −β₃·fal(ẑ₁−y, α₂, δ)              ← 總擾動估測

控制律：u = (u₀ − ẑ₃) / b₀
        u₀ = k_p(r − ẑ₁) − k_d·ẑ₂

頻寬參數化（高志強）：β₁=3ω₀, β₂=3ω₀², β₃=ω₀³
API 設計
// js/control/adrc.js
designADRC({ order=2, ω₀, ω_c, b₀, plantOrder })
   → { observer, controller, params: {β, k}, bandwidth }
simulateADRC(plant, adrc, ref, dt, T)
   → { t, y, u, ẑ }
tuneADRCBandwidth(plant, specs) → { ω₀, ω_c, predictedPM }
檔案
js/control/adrc.js (新增 ~200 行) scripts/verify_a1_adrc.mjs (新增) index.html (UI 面板) js/app.js (路由)
驗收條件
  • ✓ 一階模型 + 階躍擾動 → 穩態誤差 < 1e-3
  • ✓ 與 LQR+Kalman 對比：擾動抑制不遜色
  • ✓ 參數變動 ±50% 仍穩定
  • ✓ fal 函數連續可微（分段點 C¹）
依賴

既存 observer 模組

A2

ILC — Iterative Learning Control

HIGH 2 天
目的

重複任務之循環學習（半導體曝光機、機械臂、AGV）。目前 ControlStudio 完全缺。

演算法核心
P-type：     u_{k+1}(t) = u_k(t) + Γ·e_k(t)
PD-type：    u_{k+1}(t) = u_k(t) + Γ_p·e_k(t) + Γ_d·de_k/dt
Norm-optimal ILC (NOILC)：
  min ||e_{k+1}||²_Q + ||u_{k+1}−u_k||²_R
  → u_{k+1} = u_k + (G'QG + R)⁻¹ G'Q·e_k     (G = lifted matrix)
收斂條件：||I − GΓ||_∞ < 1
API 設計
// js/control/ilc.js
designILC({ plant, dt, horizon, type='Ptype'|'PDtype'|'NOILC',
            Γ?, Q?, R?, ref })
   → { learningOp, lifted: { G, H } }
iterateILC(ilc, y_measured, iter)
   → { u_next, e_norm, convergence }
ilcConvergenceCheck(ilc) → { spectralRadius, monotone }
檔案
js/control/ilc.js scripts/verify_a2_ilc.mjs
驗收條件
  • ✓ 每次迭代 ||e||² 嚴格遞減（單調收斂）
  • ✓ 機械臂軌跡：10 次迭代後追蹤誤差降 90%
  • ✓ Non-minimum phase 警告或自動頻段截尾
  • ✓ 支援 batch 與 continuous 兩種模式
A3

SMC + Super-twisting — Sliding Mode Control

MED 4 天
演算法核心
First-order SMC：u = u_eq − K·sign(s),  s = c'x
Boundary layer：u_sw = −K·sat(s/Φ)
Super-twisting (2nd-order)：
  u = −k₁|s|^0.5·sign(s) + v
  v̇ = −k₂·sign(s)            (有限時間收斂、消 chatter)
增益設計：k₁ > √(2L), k₂ > L   (L = 擾動 Lipschitz 上界)
API 設計
// js/control/smc.js
designSMC({ plant, slidingSurface, disturbanceBound,
            variant='classical'|'boundaryLayer'|'superTwisting',
            Φ?, k1?, k2? })
   → { surface, controlLaw, gains, reachingTime }
simulateSMC(plant, smc, ref, t) → { t, x, u, s, sliding: bool[] }
analyzeChattering(u, dt) → { chatterIndex, ampBand }
驗收條件
  • ✓ 雙積分器 + 擾動 → 有限時間到達滑動面
  • ✓ Super-twisting u 連續、|chatterIndex| < 0.05
  • ✓ Boundary layer 厚度 vs chatter trade-off 圖
  • ✓ 自動判定滑動面可達性，給 reaching time 上界
A4

Backstepping — Strict-feedback Recursive Design

MED 5 天
演算法核心
給定 strict-feedback：
  ẋ₁ = x₂ + f₁(x₁)
  ⋮
  ẋₙ = u + fₙ(x)

遞迴 Lyapunov 設計：
  V_k = V_{k-1} + 0.5z_k²
  α_k = −z_{k-1} − k_k·z_k − f_k + α̇_{k-1}
  u   = α_n

自適應 backstepping：未知參數 θ̂ → 加 update law
API 設計
// js/control/backstepping.js
designBackstepping({ plantModel: {f, g}, gains: [k1,...,kn],
                     adaptive=false, θInit? })
   → { virtualLaws, controlLaw, lyapunov, paramUpdate? }
verifyBackstepping(design)
   → { V̇expression, isNegDef, regionOfAttraction }
驗收條件
  • ✓ 三階鏈式積分器 + 三角項：V̇ 負定
  • ✓ 與 feedback linearization 對比追蹤性能
  • ✓ 自適應版本：未知 θ 估測收斂
A5

Feedback Linearization — Input-Output / Full-State

MED 4 天
演算法核心
Input-output：找 relative degree r
  y^(r) = L_f^r·h + L_g·L_f^{r-1}·h·u
  u = (v − L_f^r·h) / (L_g·L_f^{r-1}·h)
  ⇒ y^(r) = v 為線性

全狀態：diffeomorphism z = Φ(x) → ẋ = Az + Bv
zero dynamics 穩定性檢查必須通過
API 設計
// js/control/feedback_linearization.js
computeRelativeDegree({ f, g, h, n })
   → { r, lieDerivatives: [Lh, LfLh, ...] }
designIOLinearization(model, desiredPoles)
   → { transform, controlLaw, zeroDynamics, isMinPhase }
designFullStateLinearization(model)
   → { diffeomorphism, AB, controllable }
檔案
js/control/feedback_linearization.js js/math/lie_derivative.js (新增；符號運算) scripts/verify_a5_feedback_lin.mjs
驗收條件
  • ✓ 倒單擺 linearize 後極點可任意配置
  • ✓ Van der Pol zero dynamics 不穩 → 警告
  • ✓ Relative degree 計算對稱性測試
依賴

與 A4 共用 lie_derivative.js

A6

Reset Control — Clegg / FORE

LOW 3 天
演算法核心
Clegg integrator：
  ẋ = e,  if (x·e ≥ 0)
  x⁺ = 0, if (x·e < 0)        ← reset

FORE：ẋ = −ax + be, with reset condition

Hβ 條件 (穩定性)：實現特定 LMI 即穩
頻率響應：describing function 算等效相位
→ 可超越 Bode 積分限制
API 設計
// js/control/reset_control.js
designClegg({ ki }) → resetController
designFORE({ pole, gain }) → resetController
simulateResetSys(plant, reset, ref, t)
   → { t, y, u, resetEvents: number[] }
analyzeHbeta(reset, plant) → { feasible, β, stabilityMargin }
驗收條件
  • ✓ PI 比較：reset 版本 PM 提升 ≥ 5°
  • ✓ Hβ 條件對隨機系統判定一致
  • ✓ Reset 事件頻率 vs 性能 trade-off 表
A7

Reference Governor — MOAS 預計算

MED 3 天
演算法核心
計算 Maximal Output Admissible Set (MOAS)
線上：v_k = v_{k-1} + κ·(r_k − v_{k-1})
解 max{κ : x(t) ∈ X for all t ≥ k}
constraint enforcement 而不依賴 full MPC
API 設計
// js/control/reference_governor.js
computeMOAS(plant, K, constraints, horizon)
   → { Hx, hx, isConvex }
designReferenceGov({ plant, controller, constraints })
   → governorObj
stepRG(gov, x_current, r_target) → { v_modified, κ }
驗收條件
  • ✓ Monte Carlo 100% 約束滿足
  • ✓ κ → 1 時 v → r
  • ✓ MOAS 邊界 admissibility 不變

Tier B系統識別擴充 #

補齊非線性、閉環、頻域、貝氏等識別方法族。

B1

SINDy — Sparse Identification of Nonlinear Dynamics

HIGH 3 天
演算法核心
給定 Ẋ ≈ Θ(X)·Ξ，其中 Θ(X) 為候選函數字典
  [1, x, y, z, xy, x², sin(x), ...]
求稀疏 Ξ：
  STLSQ：Ξ = lstsq(Θ, Ẋ); 反覆 zero out |Ξ_ij| < λ
  LASSO / SR3 替代

微分計算：finite-diff 或 total-variation regularized differentiation
API 設計
// js/identification/sindy.js
buildLibrary(X, { polyOrder, trig, custom })
   → Θ matrix + featureNames
sparseRegression(Θ, Ẋ, { method='STLSQ'|'LASSO', λ })
   → Ξ + activeTerms
identifyNonlinearODE(X, dt, options)
   → { equations: string[], Ξ, library, score }
檔案
js/identification/sindy.js js/math/total_variation_diff.js scripts/verify_b1_sindy.mjs
驗收條件
  • ✓ Lorenz 系統：自動回復 σ(y-x), ρx-y-xz, xy-βz 三式
  • ✓ Van der Pol：辨識 μ(1-x²)y 項
  • ✓ 加 noise (SNR=20dB) 仍能找到正確項
B2

Closed-loop Identification — Direct / Indirect / Joint I/O

MED 4 天
演算法核心
Direct：直接 ARX，偏差來自 noise correlation
Indirect：先識別 closed-loop T(s) → G = T/(K(1-T))
Joint I/O：r ⊥ noise 時，[u; y] 共同識別
推薦 Box-Jenkins (BJ) 結構（closed-loop 性質更佳）
API 設計
// js/identification/closedloop.js
identifyClosedLoop({ r, u, y, dt, controllerK,
                     method='direct'|'indirect'|'jointIO',
                     structure='ARX'|'BJ' })
   → { plant, noiseModel, bias: bool, conditionNumber }
analyzeBiasRisk(data, K) → { biasIndex, recommendation }
驗收條件
  • ✓ 已知 G、K simulate 後識別 → 與真值對比
  • ✓ Direct method low-SNR 時警告 bias 風險
B3

SRIVC — Simplified Refined IV for Continuous-time

MED 5 天
演算法核心
1. filtered LS 估初值 b̂, â
2. 迭代 IV：z = ŷ filtered through estimated A
3. Newton-step until convergence

時間導數計算：Poisson filter chain (避免 finite-diff)
不需先 c2d 即可直接識別 CT 模型
API 設計
// js/identification/srivc.js
identifyCT({ t, u, y, na, nb, λ_filter, maxIter })
   → { num, den, residual, converged }
poissonFilter(signal, λ, order) → filteredSignal
驗收條件
  • ✓ Known CT plant 模擬識別誤差 < 1%
  • ✓ SRIVC vs ARX+d2c bias 比較表
B4

GP-NARX — Gaussian Process 識別

LOW 7 天
演算法核心
y(t) ~ GP(0, K(x_t, x_t'))
  x_t = [y(t-1),...,y(t-na), u(t-1),...,u(t-nb)]
hyperparameter ML：argmax log p(y | θ) by L-BFGS
kernel：RBF, Matérn 3/2, Matérn 5/2, periodic
predictive：μ*, σ*² 含 uncertainty band
API 設計
// js/identification/gp.js
fitGP({ X, y, kernel='rbf'|'matern52', noiseVar })
   → { hyper, K_inv·y, kernelObj }
predictGP(gpModel, X_test) → { mean, var, 95CI }
optimizeHyper(initial, X, y) → optimalHyper
驗收條件
  • ✓ Mauna Loa CO₂ 子集：predictive 涵蓋 95%
  • ✓ 訓練點外 predictive variance 顯著增加
  • ✓ Hyperparameter optim 收斂
依賴

D5 L-BFGS

B5

Hammerstein / Wiener — 非線性塊狀結構

LOW 4 天
演算法核心
Hammerstein：u → [靜態 NL f(·)] → v → [linear G(z)] → y
  分離法：固定 NL → LS 解 G → 固定 G → 更新 f → 迭代

Wiener：u → [linear G(z)] → x → [靜態 NL h(·)] → y
  逆 NL 估算 + 線性識別

正交化避免共線性問題
API 設計
// js/identification/hammerstein_wiener.js
identifyHammerstein({ u, y, na, nb, nlOrder, dt })
   → { f_coeffs, G: {num,den}, fitPercent }
identifyWiener({ u, y, na, nb, nlOrder, dt }) → same
驗收條件
  • ✓ Saturation Hammerstein：辨識 saturation level ±5%
  • ✓ CT Wiener with quadratic NL：辨識係數正確
B6

MIMO FRF Identification — 頻域多輸入

MED 5 天
演算法核心
給定 multisine 輸入 U(jωk)、輸出 Y(jωk)
Ĝ(jωk) = Y(jωk) · pinv(U(jωk))
Local Polynomial Method (LPM)：減少 leakage
參數化：fit common-denominator MIMO state-space via LM
API 設計
// js/identification/freq_mimo.js
computeFRFMIMO({ U, Y, freq, method='LS'|'LPM' })
   → { G_jw: ComplexMatrix[], coherence }
fitMIMOFromFRF(frf, structure='ss', order) → { A,B,C,D }
驗收條件
  • ✓ 2×2 系統 4 個 channel 識別誤差 < 5% (mag), < 3° (phase)
  • ✓ Coherence > 0.9 視為有效資料點

Tier C狀態估測強化 #

補齊 MPC 對偶之 MHE、非高斯估測之 PF、聯合狀態-參數估測。

C1

MHE — Moving Horizon Estimation

HIGH 4 天
演算法核心
J = ||x_{k-N} − x̄||²_{P⁻¹} + Σ ||w_i||²_{Q⁻¹} + ||v_i||²_{R⁻¹}
s.t. x_{i+1} = f(x_i, u_i) + w_i
     y_i = h(x_i) + v_i
     x_i ∈ X

Arrival cost：x̄ = previous MHE estimate
warm-start：上一步解 shift
API 設計
// js/estimation/mhe.js
designMHE({ plant, horizon=N, Q, R, P0, constraints })
   → mheObj
stepMHE(mhe, y_window, u_window)
   → { x_hat, w_seq, v_seq }
依賴

重用 D1 QP active-set solver 或既存 IP QP

驗收條件
  • ✓ 線性 + Gaussian noise：MHE 收斂到 Kalman 解
  • ✓ 加狀態約束：KF 違反、MHE 滿足
  • ✓ 非線性 CSTR：vs EKF bias 較小
C2

Particle Filter — Bootstrap / SIR / Auxiliary

MED 5 天
演算法核心
Bootstrap PF：
  1. propagate: x_k^i = f(x_{k-1}^i, u_{k-1}) + w^i
  2. weight: w_k^i ∝ p(y_k | x_k^i)
  3. resample if N_eff < N/2

ESS = 1/Σw²
Resampling：systematic / multinomial / stratified
API 設計
// js/estimation/particle_filter.js
initPF({ f, h, x0Sampler, N=500, processNoise, measNoise })
   → pfObj
stepPF(pf, u, y) → { x_mean, x_cov, ESS, particles }
resample(pf, method='systematic'|'multinomial'|'stratified')
驗收條件
  • ✓ 非線性 bearing-only tracking：PF < EKF RMSE
  • ✓ 低噪聲下 ESS > N/3
  • ✓ N=100/500/2000 收斂圖
C3

Joint State-Parameter EKF (Dual EKF)

MED 3 天
演算法核心
擴增狀態：x̃ = [x; θ]
動態：ẋ = f(x, u, θ); θ̇ = 0 + w_θ
Jacobians：F̃ = [∂f/∂x, ∂f/∂θ; 0, I]
標準 EKF 應用於 x̃
注意：θ̇=0 + 小 w_θ 才能追蹤慢時變
API 設計
// js/estimation/dual_ekf.js
designDualEKF({ f, h, x0, θ0, Q_x, Q_θ, R }) → dualObj
stepDualEKF(d, u, y) → { x_hat, θ_hat, P }
驗收條件
  • ✓ 一階模型 G=K/(τs+1)：估出 K, τ
  • ✓ 慢時變 K(t) 可追蹤
  • ✓ Unobservable case 警告 (rank deficient)
C4

Smoother — Rauch-Tung-Striebel / Particle

LOW 3 天
演算法核心
RTS：
  Forward Kalman → (x̂_k|k, P_k|k)
  Backward：
    G_k = P_k|k · A' · P_{k+1|k}⁻¹
    x̂_k|N = x̂_k|k + G_k(x̂_{k+1|N} − x̂_{k+1|k})

Particle smoother：backward simulation
驗收條件
  • ✓ 線性高斯：smoothed cov ≤ filter cov
  • ✓ RTS 平均 MSE 降 ≥ 30%

Tier D最佳化核心 #

QP/MILP/SQP/L-BFGS — 為 MPC、NMPC、GP、MIMPC 等提供高效求解。

D1

Active-set QP with Warm-start

HIGH 6 天
演算法核心
min 0.5 x'Hx + g'x  s.t. Ax ≤ b, Cx = d
working set W = active inequality 子集
解 EQP (KKT 系統) → step α → update W
warm-start：以上次 W 為起點
特化：MPC 中 H, A 結構固定，可預分解 KKT
防 cycling：Bland's rule
API 設計
// js/math/qp_activeset.js
solveQPActiveSet({ H, g, A_ineq, b_ineq, A_eq?, b_eq?,
                   initialW? = null, maxIter = 100 })
   → { x, λ, μ, iter, workingSet }
preFactorKKT(H, A_eq) → factorObj   // 給 MPC 重用
驗收條件
  • ✓ 與 IP QP 同解 (tol 1e-8)
  • ✓ MPC 連續 100 步：warm-start 平均 iter < 5
  • ✓ Degenerate case Bland's rule 防 cycling
  • ✓ 對 MPC 即時性提升 3-10×
D2

OSQP-style ADMM

MED 5 天
演算法核心
QP → ADMM split：
  x update: (P + σI + A'RA)x = −q + A'R(z − λ/R)
  z update: project to box [l, u]
  λ update: λ += R(Ax − z)
indirect solver；rho 動態調整
適合嵌入式 (無除法)
驗收條件
  • ✓ Standard QP benchmark vs IP 同解
  • ✓ 大型問題 (n=1000) 比 active-set 高效
D3

SQP for NMPC — Multiple Shooting

MED 5 天
演算法核心
NLP：min f(x) s.t. g(x)≤0, h(x)=0
SQP 迭代：
  解 QP：min 0.5 d'Bd + ∇f'd s.t. ∇g·d+g≤0, ∇h·d+h=0
  line search：merit function L1 / augmented Lagrangian
  Hessian：BFGS / SR1 / exact
NMPC 結構：multiple shooting → 大稀疏 QP
驗收條件
  • ✓ 倒單擺 swing-up：SQP 比 fmincon 快 3×
  • ✓ 100 隨機 IC convergence rate ≥ 95%
D4

MILP — Branch & Bound

LOW 7 天
演算法核心
Relaxed LP → 解 → 若整數變數 fractional
  → 分支 x_i ≤ floor 或 x_i ≥ ceil
node selection：best-first / DFS
bounding：incumbent value cuts subtree
cuts：Gomory, cover, knapsack
驗收條件
  • ✓ 經典背包問題找到最優解
  • ✓ TSP 小規模 (n=10) 正確
  • ✓ Infeasible 偵測
D5

L-BFGS / Trust Region

MED 4 天
演算法核心
L-BFGS：only store m past (s, y) pairs
  H_k·g ≈ recursive two-loop algorithm

Trust region：min model_k(d) s.t. ||d|| ≤ Δ_k
  ρ = (f(x)-f(x+d)) / (m(0)-m(d))
  ρ > 0.75: Δ *= 2;  ρ < 0.25: Δ /= 4
Dogleg / Steihaug-CG subproblem
驗收條件
  • ✓ Rosenbrock：收斂步數 ≤ 50
  • ✓ Memory 使用 vs BFGS 顯著降低
D6

Mixed-Integer MPC

LOW 5 天
演算法核心
min Σ (xQx + uRu)
s.t. x_{k+1} = A_q x_k + B_q u_k   (q 是模式選擇)
     q ∈ {1, 2, ..., M}
每步解 MILP；warm-start：上一步整數解 shift
依賴

D4 MILP

驗收條件
  • ✓ Switched plant tracking：找到正確切換序列
  • ✓ Infeasibility handling 友善

Tier E數值核心修補 #

提高 CARE/Lyapunov/Sylvester 精度與條件穩定性；補齊 descriptor 系統與大型稀疏解法。

E1

Newton-CARE / Newton-DARE

HIGH 3 天
演算法核心
CARE: A'P + PA - PBR⁻¹B'P + Q = 0
Newton：給定 P_k
  Riccati operator R(P) = A'P + PA - PBR⁻¹B'P + Q
  Newton-step：解 Lyapunov:
    (A - BR⁻¹B'P_k)' ΔP + ΔP(A - BR⁻¹B'P_k) = -R(P_k)
  P_{k+1} = P_k + ΔP
quadratic convergence near solution
API 設計
// js/math/care_newton.js
careNewton(A, B, Q, R, { P0?, maxIter=20, tol=1e-12 })
   → { P, iter, residualNorm }
verifyCARE(P, A, B, Q, R) → residualNorm
驗收條件
  • ✓ 對 Schur-CARE 結果 Newton refine → 殘差降 10⁻⁴ → 10⁻¹⁵
  • ✓ Ill-conditioned R 時 Newton 比 Schur 穩
E2

Bartels-Stewart Sylvester Solver

HIGH 2 天
演算法核心
AX + XB = C
Schur(A) = U_A T_A U_A'
Schur(B) = U_B T_B U_B'
C̃ = U_A' · C · U_B
解 T_A·Y + Y·T_B = C̃ (三角形系統 column-by-column)
X = U_A · Y · U_B'
API 設計
// js/math/sylvester.js
solveSylvester(A, B, C) → X
solveLyapunovCT(A, Q) → P   // AP + PA' = -Q
solveLyapunovDT(A, Q) → P
驗收條件
  • ✓ 隨機穩定 A, B：AX + XB = C 殘差 < 1e-12
  • ✓ 與既存 Lyapunov 結果一致
依賴

既存 Schur 分解

E3

Generalized Schur (QZ) — Descriptor Systems

LOW 5 天
演算法核心
For pencil (E, A): Ex' = Ax + Bu
QZ algorithm: Q'EZ = T (upper tri), Q'AZ = S
generalized eigenvalues: λ_i = T_ii / S_ii (handles infinity)
descriptor canonical form
驗收條件
  • ✓ Regular pencil 結果與 standard Schur 一致
  • ✓ Singular E (descriptor case) 不出錯
E4

Pseudo-spectrum Computation

LOW 3 天
演算法核心
σ_ε(A) = {z : ||(zI−A)⁻¹|| ≥ 1/ε}
等價：z is eigenvalue of (A+E) for some ||E|| ≤ ε
計算 grid：σ_min(zI − A) for each z
驗收條件
  • ✓ Normal A → pseudo-spectrum = disk around eigenvalues
  • ✓ Toeplitz triangular 明顯擴張
E5

Interval Arithmetic

LOW 4 天
演算法核心
Interval = [a, b]
+, −, ×, ÷ rules
matrix interval：each entry interval
robustness：worst-case bound 而非 Monte Carlo
API 設計
// js/math/interval.js
class Interval { constructor(lo, hi); add, sub, mul, div, intersect }
intervalMatMul(IA, IB) → IM
intervalEigenvalues(IA) → { lowerBound, upperBound }
robustStabilityInterval(intervalPolynomial) → bool
驗收條件
  • ✓ Kharitonov 定理對 interval poly 一致
  • ✓ 結果包含 Monte Carlo 樣本 100%
E6

Krylov Methods — GMRES / Arnoldi

LOW 5 天
演算法核心
Arnoldi：build orthonormal basis V_m of K_m(A, b)
GMRES：min ||AV_m y − b||  over y
restart GMRES(k) for memory
preconditioning：M⁻¹A → 收斂加速
驗收條件
  • ✓ Sparse SPD 系統與 direct solve 同解
  • ✓ 大規模 (n=10000) 效率測試
E7

Condition Number Gating

HIGH 2 天
演算法核心
每次 linear solve 前：
  κ(A) = ||A|| · ||A⁻¹||
  estimate 用 LU + condest (Higham)
warning thresholds:
  κ > 1e10 → severe
  κ > 1e6  → caution
建議：scaling / regularization
API 設計
// js/math/conditioning.js
estimateCondition(A) → κ_estimate (1-norm via LU)
withConditionCheck(A, b, solveFn) → { x, κ, warning }
scaleAndSolve(A, b, method='Sinkhorn') → { x, κ_after }
注入點

注入到 matSolve、matInverse、CARE、DARE、Lyapunov 等所有 linear solve 入口。

驗收條件
  • ✓ Hilbert n=10 → 警告且 κ > 1e10
  • ✓ Scaled 後 κ 降一個數量級以上

Tier F驗證與安全 #

F1

Reachability Analysis — Zonotope / Polytope

MED 8 天
演算法核心
Zonotope: Z = {c + Σα_i g_i : |α_i|≤1}
  linear map：A·Z 仍是 zonotope (轉 g_i)
  Minkowski sum：Z1 ⊕ Z2 簡單合併 g_i
recursive: R_k = A·R_{k-1} ⊕ B·U
Polytope (H-rep, V-rep) 可選

// js/analysis/reachability.js

reachZonotope(A, B, X0, U, horizon) → Z[]
containsPoint(Z, x) → bool
plotReachSet(Z, axes) → 2D points
驗收條件
  • ✓ Unit cube + identity → 同 cube
  • ✓ 2D 隨機系統 reach set 包含 Monte Carlo 樣本
F2

Control Barrier Function (CBF)

MED 7 天
演算法核心
unsafe set ∂C = {h(x) ≤ 0}
要求 ḣ + α(h) ≥ 0 along trajectories
ḣ = ∂h/∂x · (f + g·u)

QP-CBF：
  min ||u − u_des||²
  s.t. L_f h + L_g h · u + α(h) ≥ 0

SOS programming 找 polynomial h
依賴

D1 QP + SDP (既有)

驗收條件
  • ✓ 雙積分器 + 圓形障礙：trajectory 不進入 unsafe
  • ✓ SOS-CBF feasibility 對 polynomial system
F3

LTL/CTL Formal Specification

LOW 10 天
演算法核心
LTL formula → Büchi automaton (Spot / Owl 演算法)
spec + system → product automaton
satisfaction check via emptiness
typical specs: G(¬unsafe), F(goal), G(req → F(resp))
驗收條件
  • ✓ 經典範例 G(p→Fq) 模型檢驗
  • ✓ Counterexample 重現
F4

Falsification (S-TaLiRo 風格)

MED 5 天
演算法核心
robustness ρ(spec, trajectory) ∈ ℝ
falsification：find input minimizing ρ
  優化方法：CMA-ES, simulated annealing
找到 ρ < 0 → 違規範例
API 設計
// js/verification/falsification.js

驗收條件
  • ✓ 已知 bug system：找到反例
  • ✓ Spec-bound 對 monotone trajectory 正確
F5

Importance Sampling Monte Carlo

LOW 4 天
演算法核心
estimate P(failure) where event rare
proposal distribution shifted to failure region
weighting: w = p_true / p_proposal
unbiased estimator with much lower variance
Cross-entropy method 自適應 proposal
驗收條件
  • ✓ P(X > 5) for X~N(0,1) vs analytical
  • ✓ Variance reduction ≥ 100× vs naive MC

Tier G進階 MPC #

Stochastic / Distributed / Hybrid / Move-blocking — 把 MPC 推到產業級規模。

G1

Stochastic MPC — Chance Constraints

MED 6 天
演算法核心
chance constraint: P(Ax + b ≤ 0) ≥ 1 − ε
Gaussian case → equivalent deterministic：
  Ax + b + Φ⁻¹(1-ε)·||AΣ^(1/2)|| ≤ 0
scenario approach：sample N scenarios, enforce all
guarantee：P(violation) ≤ ε with conf 1-β if N ≥ ...
驗收條件
  • ✓ Tighten test：chance vs deterministic backoff
  • ✓ Monte Carlo violation rate ≤ ε
G2

Move Blocking — 計算量減半

HIGH 2 天
演算法核心
N decision variables → M blocks (M < N)
u_k = u_block[index(k)] for k = 0..N−1
typical block lengths: [1, 2, 4, 8, 16, ...]
QP size: N → M variables, 顯著減量
API 設計
// js/control/mpc_moveblock.js
addMoveBlocking(mpcDesign, blockLengths) → mpcDesign'
驗收條件
  • ✓ block=[1,1,1,1] = standard MPC
  • ✓ block=[1,2,4]：cost 略升、計算時間降 50%
G3

Distributed MPC — Dual Decomposition / ADMM

LOW 7 天
演算法核心
dual decomposition：
  ℒ = Σ_i J_i(x_i, u_i) + λ'·(consensus constraints)
  各 agent 解 local NMPC，coordinator 更新 λ
ADMM-based 收斂條件：strongly convex
communication：only λ + boundary states
驗收條件
  • ✓ 2 子系統 coupled vs centralized MPC 同解
  • ✓ 收斂步數 vs coupling strength 圖
G4

Hybrid MPC — Switched / Logic

LOW 5 天
依賴

D6 MIMPC

演算法核心
logical → MILP via Big-M / convex hull
hybrid automaton + MPC over modes
G1 + D6 自然整合
G5

NMPC Warm-Start Strategy

MED 3 天
演算法核心
shift-and-extend：
  u_new = [u_old[1:N-1], u_old[N-1]]
  states_new = simulate(plant, u_new, x0)
collocation extension：interpolate
驗收條件
  • ✓ cold vs warm 迭代步數對比表
