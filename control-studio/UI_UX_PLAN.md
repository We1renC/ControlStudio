# ControlStudio — UI/UX 改善規劃

> 版本：2026-05-21  
> 範疇：介面美術、資訊架構、使用者體驗  
> 對應程式碼基礎：333 exported symbols / 62 modules（P28-02 API docs）

---

## 一、使用者大方向（五類核心族群）

```
ControlStudio 使用者
├── A. 控制設計者   — 以設計正確、高效的控制器為目標
├── B. 分析研究者   — 以深入理解系統行為、比較方法為目標
├── C. 學習者       — 以建立控制理論直覺、完成課題為目標
├── D. 實作工程師   — 以將設計落地、部署到真實系統為目標
└── E. 技術決策者   — 以快速審查、比較方案、輸出報告為目標
```

---

## 二、各大方向展開的中需求與功能

---

### A｜控制設計者（Control Designer）

> **核心訴求**：最短路徑從系統模型到滿足規格的控制器，參數調整要有即時回饋。

```
A. 控制設計者
├── A1. 快速建模
│   ├── 功能：系統輸入精靈（TF / SS / ZPK 選擇 + 欄位驗證）
│   ├── 功能：從步階響應資料反推系統參數（SysID 一鍵入口）
│   ├── 功能：常用範例庫（倒單擺、DC motor、質量彈簧阻尼快速套用）
│   └── 功能：模型健康診斷 badge（穩定 / 最小相位 / 可控可觀 一覽）
│
├── A2. 規格設定與追蹤
│   ├── 功能：設計規格面板（Ts、OS%、PM、GM 等欄位 + 即時合規狀態）
│   ├── 功能：Bode/根軌跡上的規格邊界線（視覺化 PM、GM 裕度）
│   └── 功能：規格通過/未過 indicator（醒目顏色 + 差距數值）
│
├── A3. 互動調參
│   ├── 功能：參數滑桿（Kp/Ki/Kd 或極零點位置）→ 圖表即時重繪
│   ├── 功能：根軌跡可拖曳極點（拖極點到目標位置，反算增益）
│   ├── 功能：Bode 圖上可直接拖移折點頻率
│   └── 功能：調參歷史紀錄（Undo/Redo，每步可命名並對比）
│
├── A4. 方法推薦
│   ├── 功能：designWizard 嵌入主流程（輸入規格 → 推薦 PID/LQR/H∞）
│   ├── 功能：各方法複雜度/適用性標籤（Complexity chip、適用場景說明）
│   └── 功能：推薦說明展開面板（說明推薦理由、需要的資訊輸入）
│
└── A5. 驗證迴路
    ├── 功能：閉迴路時域/頻域並排（Step + Bode + Nyquist 三窗格）
    ├── 功能：靈敏度函數自動繪製（S、T、KS 一鍵）
    └── 功能：穩健性報告 badge（穩定裕度 / μ 值 / 最大奇異值摘要）
```

### B｜分析研究者（Analytical Researcher）

> **核心訴求**：能深入到每個數值背後，多方法並排比較，結果可以匯出發表。

```
B. 分析研究者
├── B1. 多方案並排比較
│   ├── 功能：比較模式（選取 2–4 個設計 → 同一圖疊加，用顏色/線型區分）
│   ├── 功能：指標比較表（Ts、OS%、PM、GM、‖G‖∞、‖G‖H 並列表格）
│   └── 功能：差異 highlight（超出最佳值的格子以顏色標示）
│
├── B2. 深度數值檢視
│   ├── 功能：矩陣/向量展開面板（A、B、C、D 以表格呈現，數值可複製）
│   ├── 功能：Hankel 奇異值長條圖（直觀看截斷點）
│   ├── 功能：極零點地圖（可縮放、懸停顯示數值）
│   └── 功能：Gramian / 奇異值分解詳細頁（對稱正定性視覺化）
│
├── B3. 精細圖表控制
│   ├── 功能：軸範圍手動設定、對數/線性切換
│   ├── 功能：標記點（游標 + 讀值 readout）
│   ├── 功能：圖表主題切換（dark / light / print-friendly）
│   └── 功能：SVG/PNG 匯出（含 LTTB 降採樣選項）
│
├── B4. 資料匯入匯出
│   ├── 功能：CSV 匯入（時域量測數據 → 疊加到圖表）
│   ├── 功能：CSV/JSON 匯出（所有圖表資料一鍵）
│   ├── 功能：python-control JSON 橋接（toPythonControl / fromPythonControl）
│   └── 功能：LaTeX 符號生成（系統矩陣 → LaTeX 表示，一鍵複製）
│
└── B5. 計算透明度
    ├── 功能：計算步驟摺疊面板（Lyapunov 解 → Cholesky → SVD 過程可展開）
    ├── 功能：中間數值 tooltip（游標停在結果上，顯示輸入/方程式）
    └── 功能：警告/條件數顯示（ill-conditioned 矩陣、數值精度告警）
```

### C｜學習者（Student / Learner）

> **核心訴求**：從「不知道參數代表什麼」到「知道為什麼這樣設計」，有引導、有解釋、有互動。

```
C. 學習者
├── C1. 概念入口
│   ├── 功能：主題索引（PID / 根軌跡 / 頻域 / 狀態空間 分類入口卡片）
│   ├── 功能：每個方法附「這是什麼」摺疊說明（用語言解釋，非公式堆疊）
│   └── 功能：「從範例開始」快速按鈕（每種方法有預設範例可直接跑）
│
├── C2. 引導式工作流程
│   ├── 功能：Step wizard（建模 → 規格 → 設計 → 驗證 四步驟進度條）
│   ├── 功能：每步欄位有 hint 氣泡（解釋參數含義、常用範圍）
│   └── 功能：常見錯誤提示（輸入不穩定系統 → 提示並給出建議）
│
├── C3. 互動視覺化教學
│   ├── 功能：極點動畫（拖移極點 → 即時看時域響應如何改變）
│   ├── 功能：參數敏感度圖（單一參數掃描 → 輸出指標變化曲線）
│   ├── 功能：相平面互動（初始條件可拖曳，軌跡即時重繪）
│   └── 功能：Nyquist 動畫（頻率從 0→∞ 掃描，游標跟著走）
│
├── C4. 學習進度與書籤
│   ├── 功能：設計草稿自動儲存（localStorage，刷新不遺失）
│   ├── 功能：書籤/筆記系統（在任意頁面附加文字備註）
│   └── 功能：已完成步驟 badge（完成設計後顯示通過狀態）
│
└── C5. 輸出作業
    ├── 功能：設計報告一鍵生成（generateDesignReport → 符合課堂繳交格式）
    ├── 功能：截圖工具（選取區域 → SVG/PNG）
    └── 功能：結果摘要卡（可分享的 HTML snippet，含主要指標）
```

### D｜實作工程師（Implementation Engineer）

> **核心訴求**：設計完畢後，快速產出可信任的程式碼，並知道在目標平台上的注意事項。

```
D. 實作工程師
├── D1. 程式碼生成
│   ├── 功能：語言選擇器（C / Python / MATLAB / Simulink stub）
│   ├── 功能：程式碼預覽面板（syntax highlight + 行號）
│   ├── 功能：一鍵複製 / 下載 .c/.py/.m 檔
│   └── 功能：生成選項（固定點/浮點、頭文件樣式、命名慣例）
│
├── D2. 離散化工具
│   ├── 功能：取樣時間建議（根據頻寬自動估計 Nyquist 需求）
│   ├── 功能：離散化方法比較（ZOH / Tustin / 向前/向後差分並排）
│   ├── 功能：離散後穩定性自動驗證（單位圓內極點 badge）
│   └── 功能：z-domain Bode 圖（與連續系統疊加比較）
│
├── D3. 即時性評估
│   ├── 功能：浮點運算量估算（FLOP count for MPC / Kalman / H∞）
│   ├── 功能：記憶體佔用估算（矩陣尺寸 → bytes）
│   └── 功能：平台標籤（STM32 / Raspberry Pi / PC 適用性 chip）
│
├── D4. 驗證套件
│   ├── 功能：單元測試樣板生成（pytest / GTest stub，含輸入輸出範例）
│   ├── 功能：數值精度比對（JS 計算 vs python-control 結果 diff 顯示）
│   └── 功能：HIL 資料格式（CSV 輸入 → 模擬輸出對比）
│
└── D5. 整合說明
    ├── 功能：初始化說明（程式碼使用前的狀態初始化說明段落）
    ├── 功能：反饋迴路接線圖（自動生成 ASCII 或 SVG 方塊圖）
    └── 功能：警告清單（warnings[] 顯示在程式碼旁，如「增益裕度不足」）
```

### E｜技術決策者（Technical Decision Maker）

> **核心訴求**：不需要親自調參，但需要快速看懂設計品質、批准方案、留下紀錄。

```
E. 技術決策者
├── E1. 儀表板總覽
│   ├── 功能：專案摘要卡（系統名稱、控制器類型、主要指標一覽）
│   ├── 功能：規格合規狀態（全部通過 / N 項未達標 + 差距比率）
│   └── 功能：設計時間軸（版本歷史，每個版本的關鍵指標趨勢）
│
├── E2. 方案比較報表
│   ├── 功能：多方案評分矩陣（效能 / 穩健性 / 複雜度 三軸評分）
│   ├── 功能：雷達圖（各設計的多維度指標視覺化）
│   └── 功能：推薦標籤（最佳穩健 / 最快響應 / 最低複雜度 醒目標示）
│
├── E3. 報告輸出
│   ├── 功能：HTML 設計報告（generateDesignReport 含封面、摘要、圖表）
│   ├── 功能：PDF 列印樣式（print-friendly CSS，圖表不截斷）
│   └── 功能：機密等級浮水印（可選：在報告上加蓋 DRAFT / CONFIDENTIAL）
│
└── E4. 稽核追蹤
    ├── 功能：設計決策日誌（每次參數變更自動記錄 + 可加評語）
    ├── 功能：規格來源標記（每條規格可連結到需求文件 ID）
    └── 功能：電子簽核欄位（設計者 / 審核者 / 日期 欄位，嵌入報告）
```

## 三、架構改善層面（跨族群共通）

```
全局架構
├── F1. 資訊架構重組
│   ├── 現狀：功能散落在各 js/ 模組，無統一入口
│   ├── 改善：左側導覽分四層（建模 → 設計 → 分析 → 輸出）
│   └── 改善：頂部 Context Bar（目前系統名稱 + 控制器類型 + 規格狀態）
│
├── F2. 工作區配置
│   ├── 改善：可分割工作區（左圖 + 右參數，或上下並排）
│   ├── 改善：標籤頁系統（多個設計並行開啟，可切換）
│   └── 改善：全螢幕圖表模式（按下展開鍵，圖表佔滿視窗）
│
├── F3. 狀態可見性
│   ├── 改善：全域狀態列（計算中 spinner / 完成 ✓ / 警告 ⚠ / 錯誤 ✗）
│   ├── 改善：Dirty 標記（有未儲存變更時標示 •）
│   └── 改善：計算進度條（MPC / H∞ 等較慢計算顯示百分比）
│
├── F4. 主題與視覺
│   ├── 改善：Light / Dark / Print 三主題切換（已有 theme.js 基礎）
│   ├── 改善：品牌色一致性（設計 token 延伸至所有元件）
│   ├── 改善：圖表配色系統（8 色循環，對比度 WCAG AA 以上）
│   └── 改善：動態減少模式支援（prefers-reduced-motion）
│
└── F5. 可及性（Accessibility）
    ├── 改善：完整鍵盤導覽（Tab 順序、Esc 關閉、方向鍵操作圖表）
    ├── 改善：螢幕閱讀器支援（aria-live 更新、圖表文字替代描述）
    ├── 改善：高對比模式（已有 contrastRatio / meetsWCAG 基礎）
    └── 改善：Skip link 到主內容（已有 skipLink() 元件）
```
